# 综述论文收集：4人并行任务方案

## 一、背景与目标

**综述主题**：LLM + 开源组件/SCA 安全、代码审计、可达性验证、误报消减

**总目标**：构建一个约 50 篇高质量、可核实论文的综述文献池，覆盖 6 个子方向。

**已有基础**：

* 已核实 24 篇（有 arXiv ID/DOI）

* 待核实 5 篇（LLM4SA、LLMDFA、GRACE、LineVul、V-SZZ，需补 ID）

* **本轮新增目标**：每人收集 12–15 篇高质量的、已有论文未覆盖的新论文，4 人合计新增约 50 篇

**时间范围**：2022–2026，主体 2024–2026

***

## 二、6 子方向 Taxonomy

## 四、真实性铁律（必须遵守）

1. **每篇论文必须有可解析的 arXiv ID 或 DOI**——无 ID 的不收
2. **每条 ID 必须亲自在 arxiv.org 或 dblp.uni-trier.de 核实**——能打开论文页面才算核实通过
3. **如实记录链接**：arxiv.org/abs/XXXX.XXXXX 或 doi.org/XX.XXX/XXXX
4. **摘要/贡献描述必须来自本人阅读论文后的理解**，禁止复制搜索引擎摘要
5. **拿不准的论文放进"待核实区"**，标注未核实原因
6. **宁可少报，绝不杜撰**

## 五、统一收集格式

每人提交一份 CSV 或 Markdown 表格，包含以下字段：

| 字段         | 说明                   | 示例                                  |
| ---------- | -------------------- | ----------------------------------- |
| `论文简称`     | 简短好记的称呼              | VFArche                             |

| `arXiv ID` | arxiv.org/abs/ 后的 ID | 2506.18050                          |
| `DOI`      | 如有                   | 10.1145/xxxxxxx                     |
| `年份`       | 发表/预印年份              | 2025                                |
| `会议/期刊`    | 如适用                  | ICSE, USENIX, NeurIPS               |
| `所属方向`     | ①–⑥ 编号               | ②                                   |
| `核心贡献`     | 2–3 句话概括方法创新         | 提出版本+函数双维度定位...                     |
| `与综述相关性`   | 为什么本文对综述有用           | 提供了 SCA 版本定位的新范式...                 |
| `核实状态`     | 已核实/待核实              | 已核实                                 |
| `核实链接`     | 你点开过的论文页面 URL        | <https://arxiv.org/abs/2506.18050>  |
| `备注`       | 特殊说明                 | 与 Vercation 方法对比可写一段                |

## 六、搜索策略与渠道

优先级由高到低：

1. **arXiv**：用关键词搜索 cs.SE、cs.CR、cs.AI 分类，按时间倒序
2. **dblp**：按作者/关键词追踪，适合补会议论文
3. **Google Scholar**：引文追踪——从已核实论文的"被引"和"引用"找关联论文
4. **Semantic Scholar**：推荐相似论文功能
5. **顶会 proceedings**：ICSE、FSE、ASE、ISSTA、USENIX Security、NDSS、CCS、S\&P 2024–2026 的程序册

**推荐搜索式**（每人根据方向调整）：

```
(LLM OR "large language model") AND (vulnerability OR "code audit" OR "static analysis" OR SAST OR SCA)
("false positive" OR "false alarm") AND (LLM OR "large language model") AND ("static analysis")
("call graph" OR CPG OR "code property graph") AND (LLM OR "large language model")
("software composition analysis" OR SCA) AND (reachability OR "version matching")
```

## 七、收录原则

**优先收录：**

* 有清晰方法创新且有实验数据的

* 与 Java/JVM 生态相关的

* 涉及静态分析+LLM 结合的

* 来自顶会/顶刊的（ICSE、FSE、ASE、ISSTA、USENIX、NDSS、S\&P、TSE、TOSEM）

* 2024–2026 年的最新工作

**排除（除非特别相关）：**

* 纯 C/C++ 且方法无法迁移到 Java 的

* 仅做漏洞检测不做可达性/误报分析的

* 无实证评估的 position paper / short paper

* 非英文论文

***

## 八、去重与协作规则

1. **开工前**：每人建一个共享文档（如飞书/腾讯文档），边搜边写，实时可见，避免撞车
2. **收集期间**：搜到疑似别人方向的论文时，先看一眼他人的共享文档，若已收录则跳过
3. **第4–5天**：D 同学做一次交叉检查，补各方向可能遗漏的论文，目标补齐到总计 50 篇
4. **第6天**：团队集中过一遍，确认所有 ID 可核实
