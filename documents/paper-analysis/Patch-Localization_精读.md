# Patch-Localization (LLM-SPL) 精读笔记 (v2)

> arXiv:2409.06816 | Yu et al. | WPI + Indiana University + Microsoft | 2024

---

## A. 大白话总结（3-5句）

LLM-SPL 解决的是"给定一个 CVE 漏洞编号，从开源项目的众多 commit 里找到真正修复它的那个 patch"这个安全问题。作者发现两个痛点：① CVE 描述和 commit 内容往往没有直接关联（比如一个 commit 只有"修复内存问题"几个字，很难看出和哪个 CVE 有关）；② 一个漏洞可能被多个 commit 协同修复（1-N 问题），现有方法只考虑一对一。他们的方案是：先用传统推荐模型（VCMatch）排个初榜，再让 LLM（GPT-3.5）只对 top 100 的 commit 做精判（判断是否真修复了该 CVE），以及构建 commit 之间的协作关系图，最后通过联合学习模型融合这三类特征输出终榜。用这个"LLM 只在关键处介入"的策略，把直接硬用 LLM 需要的 62 万美元/百年工期降到了 880 美元/3 天。

## B. 术语卡片（3-5个）

- **SPL (Security Patch Localization)**: 从 OSS 仓库中定位出修复特定 CVE 的 commit 的任务。
- **1-N CVE**: 需要多个 patches 联合修复才能完全解决的漏洞（占全部 CVE 的 21.04%）。
- **F1 (LLM-judged CVE-commit relevance)**: LLM 判断一个 commit 是否为某 CVE patch 的二元特征向量。
- **F2 (LLM-endorsed inter-commit relationship graph)**: LLM 构建的 commit 间协作关系图（边表示两个 commit 共同修复同个漏洞）。
- **Pseudo-relevance feedback (PRF)**: 先排初榜，只把 top-k 的结果送给 LLM 做标注，用这些标注反馈优化模型排序。

## C. 核心知识点（3-5个，只留硬数字和结论）

1. **直接 LLM 做 SPL 不可行**: 召回率 87% 但假阳率 >85%（每 10 个预测有 8+ 错误），且成本高达 62 万美元 + 百年工期。
2. **LLM-SPL 全量数据集表现**: R@10 = 92.74%，N@10 = 87.33%，M@10 = 2.34（平均只看 2.34 个 commit 就能找到全部 patch）。
3. **1-N 场景提升显著**: 对比 VCMatch，R@10 从 60.30% → 83.13%（+22.83%），N@10 从 60.99% → 80.40%（+19.41%），人工审核量减少 25%+。
4. **成本控制的核心**: 用 LLM 反馈（F1/F2）而非 LLM 直接判决 —— k=100（初榜）→ k=8（终榜），总成本从 62 万美元降至 880 美元。
5. **21.04% 的 CVE 需要多个 patch 协同修复**，但现有 SPL 方法几乎全部忽略此问题。

## D. 综述用途

### ① 双轴坐标

- **横轴（补静态分析的短板）**: 版本与可达性的鸿沟 —— 补丁定位本质上回答"哪个 commit 修复了此漏洞"，是版本/补丁级别而非函数/可达性级别。
- **纵轴（健全性代价）**: 无健全性保证 —— 推荐排序问题，目标是"找全找对"，不承诺零漏报。

### ② 可引硬数字

- R@10 = 92.74%, N@10 = 87.33%, M@10 = 2.34。
- 1-N 场景 R@10 从 60.30% → 83.13%。
- 成本从 62 万 → 880 USD。
- 21.04% CVE 需多 patch 协同修复。

### ③ 方法演进谱系

```
VCMatch (SOTA SPL, 特征工程+推荐模型)
  └── LLM-SPL (VCMatch + LLM反馈F1/F2 + 联合学习)
       ├── 前驱: PatchScout [36], FixFinder [14]
       └── 思路可推广: LLM-feedback 范式适用于其他SE任务
```

核心创新不是用 LLM 替代系统，而是让 LLM 做"特征增强"——只负责判断相关性、构建关系图，排序仍由传统模型完成。

### ④ Gap 贡献

- **暴露 gap**: 1-N 多 patch 协同修复是现有 SPL 盲区；LLM 高假阳率使其不能直接部署。
- **对双轴框架的意义**: 提供了"LLM 作为特征增强器而非决策器"的参考架构；其对成本控制的方案（PRF + top-k 限制）对小模型部署议程有启发。
