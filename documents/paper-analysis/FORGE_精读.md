# FORGE 精读笔记

**论文标题**: FORGE: Multi-Agent Graduated Exploitation and Detection Engineering

**作者**: Farooq Shaikh (Dynatrace, Vienna, Austria)

**arXiv**: 2606.03453 (2026-06)

**方向**: ⑤ LLM 智能体定位与可利用性验证

---

## A. 大白话总结（3-5 句）

FORGE 是一个把 CVE 元数据变成可衡量的利用深度 + 检测规则的多智能体系统。传统 PoC 生成只有"成功/失败"二元结果，FORGE 用 L0-L3 四级分级来衡量利用进展，失败的利用尝试也能留下有用信息。系统有 5 个 Agent 流水线工作：Intel Agent 读取 CVE/NVD 信息 → Generator Agent 自动生成包含漏洞的最小应用 → Planner Agent 制定攻击计划 → Exploit Agent 执行攻击（工具门控：前 3 轮只侦查，第 4 轮才能利用）→ Detector Agent 收集 OpenTelemetry 执行痕迹生成 Sigma/Snort 检测规则。在 603 个 CVE 上测试，67.8% 达到 L1+ 利用，每 CVE 成本 $1.50。更重要的是，所有 EPSS 和 CVSS 分数段上的利用成功率都大约 68%，说明元数据优先级和实际代码级利用能力是正交的。

## B. 术语卡片（3-5 个）

- **Graduated Exploitation Taxonomy (四级分级利用分类法)**: L0=无证据，L1=触发漏洞（如路径遍历返回文件内容），L2=成功利用（数据泄露/代码执行证据），L3=完全攻陷（RCE 有输出/完整数据转储/持久化）。一经达到的等级不会后退。XSS/CSRF/路径遍历等 CWE 被 oracle 结构性上限约束为最高 L2。
- **LLM-Primary Oracle (LLM 主导裁决器)**: 每轮 Exploit Agent 调用工具后，oracle 接收工具请求/响应、CWE 特定证据标准、应用源码、服务端快照，输出结构化的等级裁决（含置信度 0-1 和推理过程）。Agent 自评分数被忽略。使用独立模型（GPT-5 Mini）防止循环作弊。
- **Targeted Application Generation (目标应用生成)**: Generator Agent 从 CVE 元数据中提取受影响包名 + 版本，在包注册中心（npm/PyPI/RubyGems/Packagist）验证版本后，自动生成仅包含该漏洞的最小 Web 应用。不克隆原项目工程，避免依赖复杂构建环境。含 Actor-Critic 验证循环和 AppVerifier 反伪造检测。
- **Tool Gating (工具门控)**: Exploit Agent 的前 3 次 LLM 调用只允许使用侦查类工具，第 4 次之后开放所有利用工具。防止 Agent 在充分了解攻击面之前就仓促利用。
- **Knowledge Architecture (知识架构)**: 5 个知识库跨 CVE 累积经验：(1) 四级情报库（带 90 天置信衰减），(2) CWE 构建配方库，(3) Cookbook 自动蒸馏技巧库（每类 20 条上限），(4) 包经验库（按包名/生态索引，含情节/语义/程序三层次），(5) 检测知识库（按 CWE 分四层检测规则参考）。

## C. 核心知识点（3-5 个，只留结论和硬数字）

1. **利用深度数据**: 603 个 CVE 中，424 个（70.3%）通过生成+部署，409 个（67.8%）达到 L1+。其中 L0 15（3.5%）、L1 81（19.1%）、L2 295（69.6%）、L3 33（7.8%）。3.5% 的 L0 率意味着在成功生成的应用中 96.5% 至少达到 L1。二元指标把 91.9% 的 L1-L2 结果全部压缩成"成功"，丢失了粒度信息。
2. **与优先级指标的关联**: EPSS 和 CVSS 与利用深度无显著秩相关（Spearman ρ=0.062, p=0.13 和 ρ=0.016, p=0.71）。各 EPSS 区间利用率为 68% 左右（<0.001: 68.6%, 0.001-0.01: 68.0%, 0.01-0.1: 67.7%, ≥0.1: 62.5%）。这证明元数据优先级（EPSS/CVSS）与代码级模式可达性是正交属性。
3. **检测规则质量**: L2+ 利用生成的规则比 L1 有更高的 span 归一化 grounding（0.307 vs 0.175, p=0.035），即每收集一个 span 携带更多检测相关证据。生成的 Snort 规则 93.4% 对合成良性 HTTP 流量产生零误报。
4. **成本**: 每 CVE 平均 $1.50（vs CVE-GENIE 的 $2.77），Exploit Agent 占 45.3% 成本。使用 Claude Sonnet 4.5 做 Agent，GPT-5 Mini 做 oracle critic。
5. **知识迁移**: 包级别跨 CVE 构建失败的 48.0% 被在后续同包 CVE 中避免；CWE 级别利用死胡同的 95.5% 被在后续同 CWE 的 CVE 中避免。知识库注入审计确认经验签名在 Agent prompt 中确实存在。

## D. 综述用途

### ① taxonomy 归类 + 双轴坐标

- **子方向**: ⑤ LLM 智能体定位与可利用性验证
- **横轴（补偿目标）**: 模式不精确（误报消减？不完全是）+ 版本与可达性鸿沟（接近）—— FORGE 的核心问题不是消减静态分析误报，而是从 CVE 元数据出发自动验证"这个漏洞到底能不能利用"，属于"可达性验证 + 可利用性分级"的新维度。
- **纵轴（健全性代价）**: 声称两者兼顾（但非严格）—— FORGE 的四级分类法实际上在同时消减误报（L0-L3 分级替代二元判决，减少虚警）和漏报（L1 捕捉到"触发但不能完整利用"的有用中间态），但没有零漏报保证。
- **部署就绪度**: 中等。依赖云端大模型（Claude Sonnet 4.5 + GPT-5 Mini）；可解释性好（结构化 oracle 裁决 + 知识库累积）；验证在 603 个真实 CVE 上完成。但生成的"最小应用"缺乏生产环境复杂性（WAF/认证/限流），L2/L3 是"模式级可达性"而非"部署级风险"。

### ② 可引硬数字（标注来源上下文）

- "603 个 CVE 中 67.8%（409/603）达到 L1+，每 CVE 成本 $1.50，覆盖 8 种语言、187 个 CWE 类型。"（摘要 + Section 4.2）
- "EPSS 各区间利用率为~68%（ρ=0.062, p=0.13），CVSS 为~68%（ρ=0.016, p=0.71），元数据分数与利用深度无显著相关。"（Section 4.3, Figure 2b）
- "L2+ 检测规则 span 归一化 grounding 0.307 显著优于 L1 的 0.175 (p=0.035)；Snort 规则 93.4% 零误报。"（Section 4.4, Table 3）
- "Oracle 验证：92.3% VALID（95% CI [84.8%, 96.9%]）。"（Section 4.1）
- "CWE 级利用死胡同避免率 95.5%（95% CI [94.7%, 96.3%]）；包级构建失败避免率 48.0%。"（Section 4.5）

### ③ 在方法演进谱系的位置

FORGE 处在"LLM Agent 自动漏洞复现"这条线上，核心参考 CVE-GENIE（Ullah 2025, arXiv:2509.01835，51% 成功率，$2.77/CVE，需克隆原项目）。FORGE 的关键改进是：(1) 用"生成最小应用"替代"克隆原项目"，降低了 46% 成本并恢复了 CVE-GENIE 失败的 63.4%；(2) 用四级分级替换二元判决，保留了部分进展的信息；(3) 跨 CVE 知识库实现经验累积。FORGE 还连接了三个此前孤立的社区（PoC生成/优先级验证/检测规则工程），这是其最独特的位置。

### ④ 暴露的局限/对 gap 段的贡献

- **最小应用保真度有限**: 无 WAF/认证/限流/多级架构，L2+ 结果是"模式级可达性"，不反映真实部署环境 → 与 Agenda 3（统一评测框架）相关：需要区分"实验室可利用性"和"生产可利用性"。
- **仅 2024-2025 开源 CVE**: 不覆盖闭源软件、嵌入式系统、多组件漏洞 → 跨语言跨生态覆盖仍是 gap。
- **Oracle 仍有假阳性**: 92.3% VALID 意味着 7.7% 的 non-VALID，审计发现一个案例是授权绕过被误判为 L3 → 自动化裁决的可靠性仍是开放问题。
- **不处理 Java 框架动态分发**: 生成的是最小应用，不涉及 Spring/反射/SPI 等 Java 特有动态分发断裂 → 与 Agenda 1 和 Agenda 4 直接相关。
- **依赖商用闭源 LLM（Claude Sonnet 4.5 + GPT-5 Mini）**: 不支持离线部署 → 与 Agenda 2 相关。
- **可解释证据链有限**: Oracle 输出等级+置信度+推理，但未涉及合规审计所需的结构化证据链 → 与 Agenda 5 相关。
