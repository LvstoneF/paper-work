# 保守分析（Complementary Security Analysis）精读笔记

> arXiv: 2506.16899v2 | 方向 ③ LLM 消减误报 | ★ 必引必区分 | 2025（IEEE ISI 2025）
> Wagner et al., "Towards Effective Complementary Security Analysis using Large Language Models"

---

## A. 大白话总结

SAST 工具（不运行程序、只扫源码找漏洞的安检机）有个老毛病：报一堆假警报，30%~100% 都是误报，安全专家得一个个手工排查，累死。这篇论文想让大模型当「二审法官」——SAST 报完之后，把每条告警连同代码丢给 LLM，让它一步步推理这到底是真漏洞还是误报，最后打个 0~10 分的信心分。**关键约束是「保守」：宁可放过几个误报不删，也绝不能把一个真漏洞当误报删掉**（漏一个真的就可能酿成大事故）。结果发现：用 Chain-of-Thought（让模型分步推理）+ Self-Consistency（同一题让模型答 5 遍取共识）这两招，最好的模型能在「一个真漏洞都不漏」的前提下，干掉 62.5% 的误报；多个模型投票组合起来能干掉 78.9%。最反直觉的是——**个头最大的模型（如 Llama 3.1 70B）反而做不到保守**，它误报删得最狠（86.7%）却把真漏洞也误删了，直接出局。

## B. 术语卡片

> 「保守分析 / 零漏报保守策略」→ 见词典：零漏报保守策略（Fail-Safe / Conservative）。本文就是给这个策略正式命名（conservative analysis）并量化验证可行性的论文。
> SAST / 误报 FP / 漏报 FN / CWE / In-Context Learning → 均见词典。

### 术语 1：Chain-of-Thought (CoT，思维链)
- **大白话**：让模型别直接给答案，而是先把推理过程一步步写出来，最后再下结论。"Let's think step by step" 就是它的咒语。
- **在本文中的作用**：本文要求 LLM 先输出一段结构化推理（为什么这是/不是漏洞），再给 0~10 的信心分——分步推理比直接判断的误报检出率明显更高。
- **打个比方**：考数学应用题，老师要求"写出解题步骤"而不是只填答案——写步骤的学生不仅更容易答对，还能让老师看出他是真懂还是蒙的。

### 术语 2：Self-Consistency (SC，自一致性)
- **大白话**：同一个问题，让模型用稍高的随机性（temperature=0.7）独立回答好几遍（本文是 5 遍），然后综合这几次结果取共识，而不是只信一次回答。
- **在本文中的作用**：在 CoT 基础上叠加 SC（跑 5 遍），把保守模式下的误报检出率又往上抬了几个百分点——代价是时间、成本、能耗都翻 5 倍。
- **打个比方**：重要决定不靠一个人拍板，而是找 5 个专家各自独立判断再开会取共识——单个专家可能犯糊涂，但 5 个人一起出错的概率小得多。

### 术语 3：阈值 + 信心分机制 (Threshold / Decision Score)
- **大白话**：模型对每条告警打 0~10 分（10=肯定是真漏洞，0=肯定是误报，5=拿不准）。设一个阈值（比如 6），分数低于阈值的才敢标成"误报"删掉。阈值越高，删得越保守。
- **在本文中的作用**：这是实现"保守"的旋钮——找到那个"既不漏任何真漏洞、又能删最多误报"的最高阈值。每个模型的最佳阈值还不一样（GPT-4o 是 6，Phi-4 是 2）。
- **打个比方**：医院设定"癌症筛查指标超过多少才判健康可以回家"——指标卡得越严，越不会放走真病人，但健康人也得多做几项检查。

## C. 核心知识点

### 知识点 1：「保守分析」被正式定义并证明可行
论文把"删误报但绝不漏真漏洞"这件事命名为 conservative analysis，硬约束是 **TPR（真阳性召回率）= 100%**（一个真漏洞都不能漏）。在 OWASP Benchmark 测试集上，**有 6 个模型做到了零漏报前提下删误报**：GPT-4o、Phi-4、Qwen2.5-32B、Qwen2.5-72B、Phi-3 Medium 128k、Qwen2.5-Coder-7B。这证明了"让 LLM 当保守二审"在原理上行得通——这是本文最核心的贡献，也是我们综述方向③的理论基石。

> 📊 **关键数字**：误报基线 30~100%；OWASP 测试集 403 条（275 真 + 128 误报）；6/约20 个模型达成零漏报保守。

### 知识点 2：反直觉——最大的模型不一定最保守
个头最大的模型反而翻车。**Llama 3.1 70B 删了 86.7%（111/128）的误报，检出率全场最高，但它同时误删了真漏洞，不满足保守约束，直接出局**；部分 Gemini 变体同样失败。能力强 ≠ 保守。这说明"零漏报"是一个独立于"检出能力"的维度，必须单独验证——这一点对我们强调 D3（零漏报保守）作为独立评估维度极有说服力。

> 📊 **关键数字**：Llama 3.1 70B 检出 86.7% 误报却漏报出局；Phi-4 在最低阈值下保守检出 46.1%（59/128）。

### 知识点 3：CoT + Self-Consistency 是主力增益，且无需微调
全程不微调，纯靠提示工程。**3-shot CoT 最有效**（3-shot 和 5-shot 效果持平但省 35% token），叠加 **Self-Consistency（跑 5 遍取共识）**后，保守检出率进一步提升。最佳单模型在零漏报前提下删 **62.5% 误报，F2=96.8%，Precision=87.8%**。亮点是用了开源模型（Qwen 系列、Phi 系列），可私有化部署，适合隐私敏感场景。

> 📊 **关键数字**：GPT-4o 与 Qwen2.5-32B 各达 62.5%（80/128）误报检出，TPR=100%，F2=96.8%，Precision=87.8%；SC 比单 CoT 提升 3.1~5.5 个百分点。

### 知识点 4：多模型集成进一步提升，但真实世界打骨折
把 3 个保守模型（GPT-4o + Qwen2.5-32B + Phi-4）的"判误报"结果取并集（每个模型都能抓到别人漏掉的独特误报），**OWASP 上误报检出从 62.5% 拉到 78.9%（F2=98.1%，Precision=91.3%）**。但换到真实项目数据集（Mnestix，5 个 SAST 工具 + 3 种语言 + 基础设施文件）就大幅缩水：**最佳单模型 Phi-4 只删 33.85%，集成也才 38.46%**，而且 **GPT-4o 在真实数据上漏掉了 1 个真漏洞、直接失去保守资格**。说明 benchmark 上的漂亮数字到真实异构环境会大幅退化。

> 📊 **关键数字**：OWASP 集成 78.9%（102/128）；真实数据集 Phi-4 单模型 33.85%（22/65）、集成 38.46%（25/65）；GPT-4o 真实数据漏报出局。

### 知识点 5：致命局限——阈值依赖标注数据
那个决定"删不删"的最佳阈值，是在**有标注的数据集上反推出来**的（知道哪些是真漏洞才能找到"零漏报的最高阈值"）。论文自己承认：**在没有标注的真实数据上，无法保证还能保持保守**。此外 few-shot 例子大多是 Java 的，泛化性存疑；OWASP 公开在网上，可能已被模型训练时见过（数据污染）。这是我们区分自身工作的关键切入点。

> 📊 **关键数字**：阈值依赖 labeled dataset；few-shot 例子 Java 为主；OWASP v1.2 可能存在训练数据污染。

## E. 5 维矩阵评分

| 维度 | 评分 | 白话理由 |
|:---|:---:|:---|
| D1 动态分发 | ○ | 只重评估 SAST 已报出的告警文本+代码片段，根本不碰调用图，更不处理反射/Spring/SPI/反序列化 |
| D2 离线小模型 | ◐ | 评估了开源小模型且主打可私有部署，但真正有效的是 GPT-4o/32B；唯一 ≤7B 且保守的 Qwen2.5-Coder-7B 检出率仅约 2%，基本没用 |
| D3 零漏报保守 | ● | 全文核心就是 conservative analysis，TPR=100% 是硬约束，明确 fail-safe 设计——这是本维度的标杆论文 |
| D4 动态验证 | ○ | 纯静态、纯文本推理，无任何运行时插桩或 PoC 执行 |
| D5 证据链 | ◐ | CoT 输出结构化推理链 + 0~10 数字信心分，可供专家复核；但不含调用路径、代码精确定位等完整可审计证据链 |

**一句话**：本文是"零漏报保守（D3）"维度的命名者和标杆，证明了纯提示工程能让 LLM 在不漏真漏洞前提下删 62.5%~78.9% 误报；但它停留在对 SAST 告警文本的静态二审，既不处理 Java 动态分发导致的调用图断裂（D1），也没有动态执行验证（D4），且其"保守"依赖标注数据反推阈值——而我们的工作正是要在**调用图层面、用离线小模型、靠动态分发可达性 + 动态验证**来提供无需标注阈值的、可证据化的保守判定。
