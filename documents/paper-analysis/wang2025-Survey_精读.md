# 精读笔记：A Contemporary Survey of Large Language Model Assisted Program Analysis

## A. 大白话总结（3-5 句）

这篇综述聚焦"LLM 辅助程序分析"，按静态分析、动态分析、混合方法三个维度系统分类现有工作。核心论点是 LLM 补了传统程序分析的短板（误报高、路径爆炸、知识门槛高），但本身也带来新问题（幻觉、非确定性输出、token 限制）。文章特别指出"将动态分析转化为静态分析"是一个新兴方向——用 LLM 在不运行代码的情况下分析运行时行为。

## B. 术语卡片

1. **程序分析（Program Analysis）**：检查程序性质以发现缺陷或漏洞的方法，分静态（不执行）和动态（执行）两大类。
2. **符号执行 + LLM 混合**：如 LLift 用 LLM 帮符号执行做路径分析和约束求解，处理传统工具无法决策的边界情况。
3. **神经符号方法（Neuro-Symbolic）**：结合 LLM 的语义理解和符号推理的形式化方法，如 IRIS 用 LLM 推断污点规范 + CodeQL 做符号分析。
4. **渗透测试智能体**：如 PentestGPT、PenHeal，用 LLM 自动化渗透测试多步骤流程，PenHeal 比 PentestGPT 检测覆盖增 31%、修复效果增 32%、成本降 46%。

## C. 核心知识点（硬数字）

| 发现 | 数据 |
|------|------|
| 论文覆盖 | 静态分析（漏洞检测/恶意软件/验证/增强）+ 动态（模糊测试/渗透测试）+ 混合 |
| LLM 用于静态增强 | IRIS：检测增 35%、误报减 80%（用 LLM 生成 CodeQL 污点规范） |
| LATTE | 二进制污点分析 + LLM 切片和提示构造 |
| LLift | LLM + 符号执行检测 Linux 内核 Use-Before-Initialization |
| SLFHunter | 静态污点分析 + LLM 检测嵌入式固件命令注入 |
| ImmI | LLM 检测内核内存管理不一致（内存泄漏/Use-After-Free） |
| 渗透测试 | PentestGPT 任务完成率比 GPT-3.5 提升 228.6% |
| PenHeal | 检测覆盖增 31%、修复效果增 32%、成本降 46% |
| 全自动 vs 半自动渗透 | 全自动成功率 21%，半自动 64% |
| 挑战 | 数据类型错误、信息丢失、变量混淆、逻辑漏洞分析困难 |
| 非确定性输出 | 同输入产生不同输出，影响可靠性 |
| 关键转向 | "动态→静态"转换：用 LLM 在不运行代码时分析运行时行为 |
