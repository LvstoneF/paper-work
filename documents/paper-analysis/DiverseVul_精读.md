# 精读笔记：DiverseVul — A New Vulnerable Source Code Dataset for Deep Learning Based Vulnerability Detection

## A. 大白话总结（3-5 句）

DiverseVul 发布了当时最大、最多样化的 C/C++ 漏洞数据集（18,945 个漏洞函数、150 个 CWE 类型、覆盖项目数超过之前所有数据集总和）。作者系统测试了 11 种模型架构（GNN、RoBERTa、GPT-2、T5 四类），发现 LLM 显著优于 GNN——但前提是有足够大体量的训练数据。最佳模型也仅 F1 47.2%、误报率 3.5%（仍过高），且对"未见项目"的泛化能力极差（F1 从 49% 暴跌至 9.4%），结论是深度学习在漏洞检测上"还没准备好"。

## B. 术语卡片

1. **未见项目泛化（Unseen Project Generalization）**：训练集和测试集按项目划分而非随机划分——模型在未见过的项目上 F1 从 49% 降到 9.4%，是部署的最大障碍。
2. **代码预训练（Code-specific Pretraining）**：CodeT5 等模型的代码专项预训练（预测变量名/函数名）比通用预训练对漏洞检测更有效。预训练任务比模型大小更重要。
3. **架构差别 vs 数据规模**：小数据下 GNN 和 LLM 差不多，但大数据下 LLM 优势显著——说明架构评估需要大数据集才可靠。

## C. 核心知识点（硬数字）

| 发现 | 数据 |
|------|------|
| 数据集规模 | 18,945 漏洞函数 + 330,492 非漏洞函数 |
| CWE 覆盖 | 150 种 |
| 来源 | 7,514 个 commit，项目数超过之前所有数据集总和 |
| 之前最大数据集 | CVEFixes（~9,900 漏洞函数） |
| 项目覆盖 | 比之前所有数据集总和多 50% |
| 最佳模型 F1（大数据） | CodeT5/NatGen：47.2 |
| ReVeal（GNN）F1（小数据） | 12.8 |
| ReVeal（GNN）F1（大数据） | 29.8 |
| 误报率 | 最佳模型 3.5%（仍过高——成千上万函数项目 → 几百 FP） |
| 未见项目 F1 | 49% → 9.4%（暴跌） |
| 数据增 5× 后 F1 提升 | 10.5 → 48.9 |
| 关键发现 | 代码预训练任务比模型大小更重要；LLM 优于 GNN 但需大数据 |
