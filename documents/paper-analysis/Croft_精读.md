# 精读笔记：Data Quality for Software Vulnerability Datasets

## A. 大白话总结（3-5 句）

这篇论文系统检查了四个主流漏洞数据集（Devign、ReVeal、BigVul、CVEfixes）的数据质量，发现极其严重的问题：20-71% 的漏洞标签不准确、17-99% 的数据存在重复。这些问题导致下游模型要么学不到有效模式，要么评估性能虚高（去重后 F1 下降最高 82%）。作者定义了五维数据质量框架（ISO/IEC 25012）：准确性、唯一性、一致性、完整性、时效性，并给出了测量方法。这是理解"为什么现有数据集不可靠"的必读论文。

## B. 术语卡片

1. **数据质量五维框架**：ISO/IEC 25012 标准定义的五个固有属性——准确性（标签正确）、唯一性（不重复）、一致性（无矛盾）、完整性（不缺失）、时效性（不过时）。
2. **标签噪音（Label Noise）**：漏洞数据集的错误标注——可能是误标"有漏洞"（假正例）或遗漏标注"漏报"（假负例）。
3. **数据泄露（Data Leakage）**：训练集和测试集之间存在重复数据（如同一漏洞出现在不同提交中），导致评估性能虚高。
4. **SVP（Software Vulnerability Prediction）**：基于学习的软件漏洞预测模型，高度依赖数据质量和数量。

## C. 核心知识点（硬数字）

| 发现 | 数据 |
|------|------|
| 分析的数据集 | Devign、ReVeal、BigVul、CVEfixes |
| 标签不准确率（真实数据集） | 20%–71% |
| 数据重复率 | 17%–99% |
| 不一致标签 | 最高 47% |
| 去重后性能下降 | 最高 82%（F1 降幅） |
| 四种标签来源 | 安全厂商（NVD）、开发者（commit）、工具（SAST）、合成 |
| 影响 | 误导模型训练、虚标基准性能、泛化失败 |
| 建议 | 系统性数据质量评估应成为 SVP 研究标配环节 |
