# 论文真实性校验提示词

> **什么时候用**：每搜到一篇候选论文，填表之前，先跑完这份校验清单。
> **铁律**：所有项目通过才能标「已核实」。任何一项不通过，挪到「待核实区」并标注原因。**宁可少一篇，绝不收假论文。**

---

## 一、校验流程总览（每篇论文走一遍）

```
找到论文 → 打开论文页面 → 逐项核对 → 全部通过? ──是→ 标「已核实」+ 填表
                                              ──否→ 标「待核实」+ 写原因
```

---

## 二、逐项核对清单（6 项，缺一不可）

对每篇候选论文，逐项打勾：

### 第 1 项：ID 可解析

- [ ] **arXiv ID**：打开 `https://arxiv.org/abs/XXXX.XXXXX`，能看到论文标题、作者、摘要
  - 常见死链：ID 输入错误（多一位少一位）、论文被撤回（页面显示 withdrawn）
- [ ] **DOI**：打开 `https://doi.org/XX.XXX/XXXX`，能跳转到论文页面
  - 常见死链：DOI 还未激活（刚发表的可能要等几天）、DOI 复制时漏了字符
- **通过标准**：浏览器地址栏显示的就是你输入的 ID，页面完整加载，能看到论文元数据

### 第 2 项：标题完全匹配

- [ ] 你记录的中文简称/英文全标题，与论文页面显示的标题一致
  - 允许大小写差异
  - 不允许：多词/少词/换词（例如页面是 "A Study of..." 你写成 "Study of..." → 不通过）
- **通过标准**：论文页面标题与你的记录逐词一致（忽略大小写）

### 第 3 项：作者吻合

- [ ] 至少有 2 位作者姓名与你记录的一致（最好全部核对）
  - 特别是第一作者和通讯作者
- **通过标准**：第一作者姓氏拼写无误，作者数量和顺序说得通

### 第 4 项：年份吻合

- [ ] arXiv 页面显示 `Submitted on [日期]` 的年份 = 你记录的年份
- [ ] 或会议/期刊论文显示的出版年份 = 你记录的年份
- **通过标准**：年份吻合，±0 年

### 第 5 项：内容相关性确认

- [ ] 读论文摘要（至少前 5 句），确认：
  - 论文主题与你分配的方向相关
  - 不是同名但不同领域的论文（例如搜漏洞检测搜出了医疗漏洞管理）
- **通过标准**：你能用自己的话说出这篇论文在解决什么问题、用了什么方法

### 第 6 项：不重复

- [ ] 在附录排除清单中搜索，确认这篇论文不在已有 29 篇中
- [ ] 在共享表的其他人 tab 里搜索，确认队友没先收了
- **通过标准**：arXiv ID / DOI 不在任何已有记录中

## 三、校验实操步骤（照做就行）

```
步骤 1：复制 arXiv ID 或 DOI
步骤 2：在新标签页打开 https://arxiv.org/abs/[你的ID] 或 https://doi.org/[你的DOI]
步骤 3：等页面完全加载
步骤 4：对照论文页面的标题，核对你的记录——逐词比对
步骤 5：扫一眼作者列表——第一作者对不对
步骤 6：看提交日期/出版年份——和你记录的是否一致
步骤 7：读一遍摘要，确认内容确实相关
步骤 8：在排除清单和共享表里搜一遍，确认不重复
步骤 9：全部通过 → 在共享表「核实状态」列填「已核实」
        任何一项不通过 → 填「待核实」，在备注列写原因
```

## 四、常见造假/错误模式（见到这些直接标「待核实」）

| 模式 | 特征 | 处理方式 |
|------|------|----------|
| **arXiv ID 能打开但是另一篇论文** | ID 对但被 arXiv 重新分配了 | 标待核实，备注「ID 映射论文不符」 |
| **只有标题搜得到，没有 arXiv/DOI** | Google Scholar 上有条目，但没正式出版也没有预印本 | 不收，除非你能找到可解析 ID |
| **论文已撤回** | arXiv 页面显示 `Withdrawn` 或 `Retracted` | 不收（除非撤回前的版本对综述有历史意义） |
| **预印本和出版版内容差异大** | arXiv v1 和会议 camera-ready 差异明显 | 两个都记录，以出版版为准 |
| **搜到的是 PPT/海报/扩展摘要** | 2–4 页的 workshop paper 而非完整论文 | 可收，但备注标注「short paper / workshop」 |
| **中文论文冒充英文** | 标题看着像英文但实际是中文期刊的英文标题翻译 | 不收（排除非英文论文） |
| **AI 生成的假论文** | 作者名看起来像真的但查不到任何其他工作、摘要逻辑不通 | 不收，备注「疑似 AI 生成，无法核实作者」 |

## 五、「待核实区」处理规则

遇到以下情况，论文可以放进「待核实区」，不丢：

| 情况 | 标注原因 | 后续处理 |
|------|----------|----------|
| arXiv 页面能打开但还没读摘要 | 「待确认内容相关性」 | 读完补标 |
| DOI 有效但需机构订阅才能读全文 | 「付费墙，仅有摘要可见」 | 通过摘要判断，仍可收录 |
| 论文刚发，DOI 还没激活（404） | 「DOI 待激活，预计 X 月可用」 | 等激活后补核实 |
| 作者名拼写不确定（例如中文拼音有多种写法） | 「作者名待确认」 | 查 dblp 确认 |
| 同一研究组发了多篇相似论文，不确定是不是同一篇 | 「疑似重复，待对比」 | 两篇都打开对比后决定 |

## 六、自检清单（提交前最后一步）

每篇论文填表前，对着这里念一遍：

1. **链接能打开吗？** → 我亲自在浏览器里打开了，页面完整
2. **标题对吗？** → 逐词比对过
3. **作者对吗？** → 第一作者没拼错
4. **年份对吗？** → 和页面上的一致
5. **内容相关吗？** → 我读了摘要，用自己的话写了核心贡献
6. **不重复吗？** → 在排除清单和队友 tab 里搜过了
7. **我有没有为了凑数而放水？** → 没有。每一项都真实通过。

## 七、校验示例

### 示例 1：通过校验 ✅

```
候选论文：2506.18050
步骤 1：打开 https://arxiv.org/abs/2506.18050
步骤 2：页面标题 "VFArche: Version-Function Aware Reachability Analysis for Open-Source Component Vulnerabilities"
步骤 3：逐词比对标题 → 一致 ✓
步骤 4：作者 Lingfeng Bao, ... → 第一作者吻合 ✓
步骤 5：Submitted on 19 Jun 2025 → 年份 2025 ✓
步骤 6：读摘要 → 做 SCA 版本+函数可达性分析 → 方向② ✓
步骤 7：在排除清单中搜索 2506.18050 → 搜不到（这是新论文）✓
步骤 8：在队友 tab 中搜索 → 没人收过 ✓
结论：6 项全部通过 → 标「已核实」
```

### 示例 2：不通过 ❌

```
候选论文：某论文 arXiv ID 2405.xxxxx
步骤 2：打开 https://arxiv.org/abs/2405.xxxxx
页面显示 "Page not found"
→ 第 1 项失败
结论：标「待核实」，备注「arXiv 链接无法打开，需确认 ID 是否正确」
```
