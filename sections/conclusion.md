# 结论

> 文献综述章节 · 总结三重贡献 + 展望

---

## 11.1 三重贡献回顾

本综述对"LLM + 开源组件安全/代码审计/漏洞可达性验证/误报消减"领域进行了系统梳理，覆盖 48 篇论文、6 个子方向，时间跨度 2022–2026。核心贡献如下：

**贡献 1：新分类法（双轴框架）。**
- **横轴** = LLM 补偿静态分析的五个短板（调用图不完整 / 模式不精确 / 污点规范缺失 / 版本-可达性鸿沟 / 数据与标注质量）
- **纵轴** = 补偿对健全性（Soundness）的代价（只剪误报 / 只补漏报 / 两者兼顾 / 零漏报保证）
- **叠加**部署就绪度评分（可编译性、云端依赖、可解释性、真实代码库验证）

该分类法使不同子方向的论文可以在同一框架内比较——例如 IRIS（方向④）和 Sifting（方向③）虽然方法完全不同，但都落在"模式不精确 + 只剪误报"区域，共享相同的健全性盲区。

**贡献 2：健全性代价系统化（现有综述盲区）。**
- 48 篇精读论文中，仅 2 篇提供接近"零漏报保证"的设计
- Sifting the Noise 是唯一系统量化了漏报代价（FN 22.25%）的工作——它暴露了问题但未解决它
- 方向⑤ 的 D4 繁荣（动态执行验证）值得肯定，但验证的是"可利用性"而非"可达性"

**贡献 3：证据驱动的 5 条研究议程（核心交付物）。**
- 从双轴矩阵的 5 个具体空白推导，每条议程都包含：问题定义、技术路线、关键挑战、评测方案、预期影响
- 最高优先：议程 1（带健全性保证的 SCA 可达性验证）——矩阵空白 (A,4) 完全空白
- 审稿人看完议程可直接用于立项

---

## 11.2 关键发现摘要

1. "静态找候选 + LLM 判语义"混合架构已成共识（67% 论文采用），纯 LLM 检测路线被实证否决
2. 双轴矩阵空白 (A,4) — 调用图断裂补偿 + 零漏报保证 — 是该领域最关键的开放问题
3. 云端大模型依赖是普遍瓶颈（90% 论文），离线部署研究严重不足
4. D1●+D2●+D3● 三元组合从未同篇出现——技术上可以组合，但无人做过
5. 数据集均不标注可达性信息——结构性缺陷限制了对 SCA 可达性方案的评测

---

## 11.3 展望

本综述推导的 5 条研究议程具有明确的可行性——每个议程所需的基础组件（静态调用图、LLM 语义推理、保守 fail-safe 策略）均有已有工作提供存在性证明。关键挑战不是"单个组件是否可行"，而是"如何将它们组合到一个健全性有保证的管线中"。

如果议程 1 得到解决，SCA 工具将从"版本比对"时代进入"可达性验证"时代——误报消减的对象从"版本不匹配"扩展到"版本匹配但不可达"，工业界 SCA 的告警洪流问题将得到根本性改善。
